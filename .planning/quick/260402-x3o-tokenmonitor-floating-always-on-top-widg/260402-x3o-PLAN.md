**FloatingWidgetWindow.swift** — Create an `@MainActor final class FloatingWidgetWindow` that manages an NSPanel for the always-on-top widget:

- Create an `NSPanel` with `styleMask: [.borderless, .nonactivatingPanel]` and `level = .floating`
- Set `isMovableByWindowBackground = true` so user can drag the pill anywhere
- Set `collectionBehavior = [.canJoinAllSpaces, .fullScreenAuxiliary]` so it appears on all desktops and in fullscreen
- Set `backgroundColor = .clear`, `isOpaque = false`, `hasShadow = true`
- Use `NSHostingView` to embed `FloatingWidgetView` as content
- Accept a `UsageWindowTracker` in init/show method, inject as `.environmentObject(tracker)` on the SwiftUI view
- Provide `show()` and `hide()` methods that set `orderFront(nil)` / `orderOut(nil)`
- Store last position in `UserDefaults` (key: `floatingWidgetPosition`) so widget reopens where user left it
- On init, restore position from UserDefaults or default to top-right of screen with 20pt margin
- Override `NSWindow` notification for moved events to save position to UserDefaults
- The panel must NOT appear in Mission Control or Expose — use `.stationary` in collectionBehavior
- The panel must NOT show in the Dock or app switcher (it's already a panel, but ensure `hidesOnDeactivate = false`)

**FloatingWidgetView.swift** — Create a compact horizontal pill-shaped SwiftUI view:

- `@EnvironmentObject var usageTracker: UsageWindowTracker`
- Layout: `HStack` with session info | small divider/icon | weekly info
- Overall shape: `Capsule()` background with dark semi-transparent material (`.ultraThinMaterial` with dark color scheme, or explicit dark background like `Color.black.opacity(0.75)`)
- Compact sizing: approximately 420x36 points (adjust as needed for content)

**Left side (Session):**
- Time left: compute from `currentWindow?.fiveHourResetTime` minus now. Format as "Xh Ym left". Use a `TimelineView(.periodic(from: .now, by: 60))` to update every minute.
- Percentage: from `fiveHourUtilization`, formatted as "XX%", color-coded:
  - Green: 0-50%, Yellow: 50-80%, Orange: 80-95%, Red: 95%+
- Trend arrow: compare current utilization to a simple heuristic — if `isLimited` show "!" in red, otherwise show "Session" label. (Trend arrows require historical data we don't have; use a static label for now, can be enhanced later.)
- Cost: from `extraUsageSpentCents` if available, format as "~$X.XX". If nil, show nothing.
- Prefix with small SF Symbol `bolt.fill` in the percentage color

**Center:**
- A small vertical divider line (1pt, white opacity 0.3)

**Right side (Weekly):**
- Time left: compute from `currentWindow?.sevenDayResetTime` minus now. Format as "Xd Yh left".
- Percentage: from `sevenDayUtilization`, same color coding as session
- Cost: show `extraUsageSpentCents` as spent this period if available, or creditBalanceCents as balance
- Prefix with small SF Symbol `calendar` in the percentage color

**Text styling:**
- All text `.font(.system(size: 11))` or `.caption2` for labels, `.system(size: 12, weight: .semibold).monospacedDigit()` for numbers
- White text (`.foregroundStyle(.white)`) since background is dark
- Use `.environment(\.colorScheme, .dark)` on the root view to ensure proper contrast

**Edge cases:**
- If `currentWindow` is nil, show "Waiting for data..." in a compact pill
- If `fiveHourResetTime` is nil, show "--" for time left
- If utilization is nil, show "--%" in gray

**Important:** Do NOT use any Liquid Glass or `.glassEffect` modifiers on this view. The floating widget uses its own dark translucent style, independent of the app's glass design.

- Add a method or NotificationCenter observer so Settings can toggle the widget:
  - Listen for `Notification.Name("toggleFloatingWidget")`
  - On receive: if enabled show, if disabled hide
  - Alternatively, expose the floatingWidget through MenubarManager

**MenubarManager.swift:**
- Add `private var floatingWidget: FloatingWidgetWindow?` property
- Add method `func observeFloatingWidget(_ widget: FloatingWidgetWindow)` that stores the reference
- Add method `func toggleFloatingWidget(enabled: Bool)` that calls `show()` or `hide()` and saves to UserDefaults

Actually, simpler approach: Since FloatingWidgetWindow reads UserDefaults and AppDelegate owns it, use NotificationCenter:
