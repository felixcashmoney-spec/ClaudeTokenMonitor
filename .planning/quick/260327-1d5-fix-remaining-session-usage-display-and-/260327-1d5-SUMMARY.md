# Quick Task 260327-1d5: Fix Remaining Session Usage Display — Summary

**One-liner:** Wired UsageWindowTracker into DashboardView via EnvironmentObject, replacing PlanUsageBanner's inline [TokenRecord] computation with the shared UsageWindow data model that averages learned limits from history.

## What Was Built

### UsageWindowTracker Service

A new `UsageWindowTracker` (`@MainActor`, `ObservableObject`) that:

1. **Polls every 10 seconds** via `Timer` to re-evaluate the usage window
2. **Detects rate-limit windows:** Finds the most recent rate-limit event and determines if we're still within the 5h window
3. **Learns token limits:** Sums tokens from the 5h window preceding each rate-limit event; averages across all historical events for accuracy
4. **Publishes `currentWindow: UsageWindow?`** — a clean data model with `tokensUsed`, `learnedLimit`, `remaining`, `usagePercent`, `resetTime`, `isLimited`

### Dashboard Integration

- **MenubarManager** gains `observeTracker(_ tracker: UsageWindowTracker)` that stores the tracker and rebuilds the popover content with `.environmentObject(tracker)` injected
- **AppDelegate** calls `manager.observeTracker(tracker)` after starting the tracker
- **DashboardView** uses `@EnvironmentObject private var usageTracker: UsageWindowTracker` and passes `usageTracker.currentWindow` to `PlanUsageBanner`
- **PlanUsageBanner** API simplified from `let tokenRecords: [TokenRecord]` to `let window: UsageWindow?` — no more inline computation
